
 ◆ HTML 기본 문서 구조

 태그, 이건 꼭 알아두세요!

 앞의 설명에서 HTML 은 웹 문서에 마크업하는 언어라고 했습니다. 어디가 이미지고 어디가 텍스트인지
 정보의 표현 방식을 정의하는 것을 마크업이라고 하는데 마크업 할 때 사용하는 약속된 표기법을 '태그(tag)'라고 합니다.
 태그는 웹 문서에 표시하려는 내용을 전달하는 언어이기 때문에 정확히 사용해야 합니다. 여기에느 몇 가지 준수사항이 존재합니다.

 1) 태그는 < 와 > 를 이용해서 구분합니다.
 웹 문서에는 태그뿐만 아니라 실제 표시될 내용까지 함께 들어 있지만 < 와 > 로 묶인 부분이 HTML 태그입니다. 예를 들어
 이미지를 삽입할 때는 image 의 약자인 <img> 태그를 사용하고 텍스트 문단을 삽입할 때는 paragraph의 약자인 <p> 태그를 사용합니다.
 어느 부분에 어떤 태그를 사용해야 하는지는 HTML 명세서에 미리 정해져 있으며, 이러한 태그들을 하나씩 공부해나가는 것이
 HTML 을 공부하는 방법입니다. 웹 브라우저는 < 와 > 사이에 있는 내용을 태그로 인식한다는 점을 꼭 알아두세요.

 2) 태그는 소문자로 씁니다.
 HTML 태그는 대, 소문자를 구분하지는 않습니다만, 표준 명세서에 따르면 태그와 태그 안에 사용하는 속성들은 모두
 소문자로 사용할 것을 권장하고 있습니다.
 <img src="images/first.jpg"> ○   <IMG src="images/first.jpg"> X

 3) 여는 태그와 닫는 태그를 정확히 입력합니다.
 대부분의 태그는 <h1></h1>, <p></p> 처럼 여는 태그와 닫는 태그가 하나의 쌍으로 이루어지며 웹 브라우저는 여는 태그에서부터
 닫는 태그까지를 태그의 적용 범위로 인식합니다. <img>, <br> 태그처럼 닫는 태그가 없는 경우도 있지만 닫는 태그가 필요한
 태그에서는 반드시 닫는 태그를 삽입해야 합니다.

 4) 적당히 들여쓰기합니다.
 HTML 태그는 기본 속성 상 여러 번 띄어쓰기를 하더라도 한 칸의 공백으로 인식합니다. 따라서 HTML 소스를 작성할 때 태그 사이의
 포함 관계에 따라 단계별로 들여 쓰더라도 웹 브라우저는 단지 한 칸으로 인식합니다. HTML 소스를 단계별로 들여쓰면 태그 간의 포함
 관계를 눈으로 쉽게 확인할 수 있어 스타일 시트나 자바스크립트를 사용하면서 문서 구조를 확인할 때 편리합니다.

 5) 태그는 속성과 함께 사용할 수 있습니다.
 HTML 태그는 단순히 태그 하나만 사용될 뿐 아니라 태그에 여러 기능을 추가하는 속성과도 함께 사용됩니다. 즉
 <태그 속성 = "속성 값", 속성="속성 값" ...> 의 형태로 사용할 수 있습니다. 각 태그마다 사용할 수 있는 속성과
 속성값들이 다 다르기 때문에 태그와 속성을 함께 익혀두어야 합니다.
 <img src="images/first.jpg" width="350" height="290" alt="시계 이미지">

 6) 포함관계를 명확히 합니다.
 HTML 태그는 한 태그 안에 다른 태그를 포함할 수 있습니다.이 때는 여는 태그와 닫는 태극 정확히 맞추어야 합니다.
 <u><i>만약 네가 나를 길들인다면</i></u>


 7) HTML 문서 기본 구조 살펴보기
 HTML 문서는 일반 문서와 달리 정해진 형식에 맞추어 내용을 입력해야 합니다. 앞에서 만든 HTML 문서를 통해
 구조를 파악해봅시다. 웹 문서는 아무리 길더라도 <!DOCTYPE> 과 <html>, <head>, <body> 네 가지 태그를 이용해
 문서의 시작과 끝을 표시합니다.

 ① <!DOCTYPE html> 현재 문서가 HTML5 언어로 작성되었다는 뜻입니다.
 ② <html> ~ </html> 웹 문서의 시작과 끝을 나타내는 태그입니다. 웹 브라우저가 <html> 태그를 만나면 </html> 까지의 소스를 읽어
 HTML 문법에 맞추어 브라우저에 표시합니다.
 ③ <head> ~ </head> 웹 브라우저가 웹 문서를 해석하기 위해 필요한 정보들을 입력하는 부분입니다. 여기에 있는 정보는
 실제 문서 내용이 아니기 때문에 브라우저 창에 표시되지는 않습니다.
  <title> - 문서 제목
  웹 브라우저 제목 표시줄에 해당됩니다. 해당 페이지를 방문하는 방문자나 검색 엔진은 제목 표시줄의 제목을 보고 페이지 전체의 내용을
  추측합니다.
  <meta> - 문자 세트를 비롯한 문서 정보
  메타데이터라고 하면 '데이터에 대한 데이터'를 말하지요. 메타 태그도 비슷합니다. <meta> 태그는 웹 브라우저 화면에는 보이지 않지만
  웹 문서에 대한 정보들을 담고 있습니다.
  <meta charset="UTF-8"> 문자 세트 지정하기: 화면에 글자를 표시할 때 어떤 인코딩 방법을 사용할지 지정하는 것입니다. 웹 서버는
  영어를 기본으로 하기 때문에 영어 이외의 문자를 화면에 표시하기 위해 미리 약속된 문자 세트를 사용해야 하는데 이 때 브라우저에게
  어떤 문자세트를 사용할 것인지 알려주는 것입니다.
  <meta name="viewport" content="width=device-width, initial-scale=1.0"> 모바일 기기 고려하기
  <meta http-equiv="X-UA-Compatible" content="ie=edge"> 익스플로러 고려하기
  <meta name="keywords" content="html5, 웹 표준"> 검색 엔진 고려하기
  <meta name="description" content="html5 를 통해 웹 표준 공부하기">

 ④ <body> ~ <body> 실제로 웹 브라우저 화면에 표시되는 내용들입니다.

 8) 웹 문서에서 특수 문자 및 특수 기호 사용하기
 HTML 소스에서 공백이나 괄호 같은 문자들을 소스 기호로 인식하지 않고 그대로 화면에 표시하려고 할 때도 특수 기호를 이용합니다.
 예를 들어, HTML 문서에서는 공백을 여러 개 넣는다고 하더라도 한 칸으로만 인식하기 때문에 여러 개의 공백을 제대로 나타내려면
 공백을 나타내는 특수 기호를 여러 개 입력해야 합니다. 또한 "" 나 < > 와 같은 기호들도 소스로 인식되지 않고 표시될 내용으로서
 지정하기 위해서는 미리 약속된 표기법을 사용해야 합니다. 이러한 표기법을 '엔티티 이름' 또는 '엔티티 기호'라고 합니다.
